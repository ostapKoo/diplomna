# Tom and Jerry Voice Assistant

## 📚 Правила документування коду
Для підтримки високої якості коду в цьому проєкті ми використовуємо стандарт **Google Style Docstrings**. Всі розробники, які долучаються до проєкту, повинні дотримуватись наступних правил:

1. **Публічні функції та класи:** Кожна публічна функція, метод або клас повинні мати Docstring.
2. **Формат:** Починайте опис з короткого речення про те, що робить функція.
3. **Параметри та повернення:** Обов'язково описуйте аргументи у блоці `Args:` та значення, що повертається, у блоці `Returns:`.
4. **Типізація:** Вказуйте типи даних параметрів (наприклад, `str`, `int`, `bool`).


Цей розділ допоможе вам розгорнути проєкт на свіжій ОС (Windows/Ubuntu).

### 1. Необхідне програмне забезпечення
- **Python 3.10 або вище** (переконайтеся, що `pip` встановлено).
- **Git** (для контролю версій).
- **FFmpeg**: Необхідний для роботи аудіо-бібліотеки `pydub` (додайте шлях до ffmpeg/bin у змінні середовища PATH).
- *(Тільки для Windows)* **C++ Build Tools**: Може знадобитися для компіляції бібліотеки `PyAudio`.

### 2. Клонування та налаштування середовища
Відкрийте термінал і виконайте наступні кроки:
```bash
# Клонування репозиторію
git clone [https://github.com/ВАШ_ЛОГІН/diplomna.git](https://github.com/ВАШ_ЛОГІН/diplomna.git)
cd diplomna

# Створення віртуального середовища
python -m venv venv

# Активація середовища (для Windows)
venv\Scripts\activate
# Для Linux/Mac: source venv/bin/activate
Встановити всі залежності( вони в файлі requirements.txt) 
pip install -r requirements.txt 
Налаштувати файл .env замінити на свої 
Запуск проєкту у режимі розробки 
Для запуску головного голосового циклу: python main.py 
Для запуску API-сервера зі Swagger UI: uvicorn api:app --reload
