# Розгортання у Production середовищі

Ця інструкція призначена для DevOps/Release інженерів для розгортання проєкту на production-машині (наприклад, сервер-термінал або локальний хаб розумного дому).

## 1. Вимоги до апаратного забезпечення
- **CPU:** Мінімум 2 ядра (1.5 GHz+).
- **Пам'ять (RAM):** 4 GB (рекомендується 8 GB для стабільної роботи Chrome та системних викликів).
- **Диск:** 500 MB вільного простору.
- **Периферія:** Підключений робочий мікрофон та динаміки (обов'язково для Application Server).

## 2. Необхідне ПЗ та Налаштування мережі
- **ОС:** Windows 10/11 або Ubuntu 22.04 Desktop.
- **Мережа:** - Відкритий **вихідний** порт 443 (HTTPS) для доступу до Gemini API та Google Speech.
  - Відкритий **вхідний** порт 8000 (TCP) для доступу до API сервера FastAPI.

## 3. Налаштування СУБД
- *Проєкт не використовує окрему СУБД.* Для збереження нотаток потрібні права на запис у директорію проєкту.

## 4. Розгортання коду (Windows Server/Desktop)
1. Авторизуйтесь на цільовій машині під сервісним акаунтом.
2. Склонуйте код у `C:\Apps\TomAndJerry`.
3. Встановіть залежності через `pip install -r requirements.txt`.
4. Налаштуйте файл `.env` із робочими (production) API ключами.
5. Налаштуйте запуск програми як фонової служби (Windows Service) за допомогою утиліти NSSM або в Планувальнику завдань (Task Scheduler) при старті системи.

## 5. Перевірка працездатності
1. Запустіть API сервер. Відкрийте в браузері `http://IP_АДРЕСА:8000/docs`. Сторінка має завантажитись.
2. Промовте команду в мікрофон сервера. Сервер має відповісти голосом.
