## Вибір інструментів
Для проєкту голосового асистента "Tom and Jerry" були обрані наступні інструменти:
- **Flake8**: Основний лінтер. Вибраний через свою швидкість, суворе слідування стандартам PEP8 та модульність.
- **Mypy**: Інструмент для статичної перевірки типів.

## Найважливіші аспекти якості для проєкту:
1. **Стиль коду (PEP8)**: Забезпечує легку читабельність коду (особливо важливо для складних логічних блоків бота).
2. **Безпека**: Виявлення невикористаних змінних та імпортів, які можуть призвести до непередбачуваної поведінки.

## Базові правила (.flake8)
- `max-line-length = 120`: Розширено стандартну довжину рядка для зручності читання довгих текстових відповідей бота.
- `exclude = venv, .git`: Виключено системні директорії для пришвидшення аналізу.

## Інструкція з запуску
Для перевірки коду виконайте в терміналі:
`flake8 .`

## Git hooks (Pre-commit)
У проєкті налаштовано фреймворк `pre-commit`. Перед кожним комітом автоматично запускаються хуки:
- Видалення зайвих пробілів (trailing-whitespace).
- Перевірка порожнього рядка в кінці файлу (end-of-file-fixer).
- Запуск лінтера Flake8.

## Статична типізація
Для статичної перевірки типів використовується `mypy`. Вона допомагає уникнути помилок `TypeError` під час виконання програми, особливо при роботі з API (Gemini, Telegram).

## Інтеграція з процесом збірки
Для локального CI/CD конвеєра створено скрипт `check_code.bat`, який послідовно запускає перевірку стилю та перевірку типів. Це імітує процес лінтингу під час збірки проєкту на сервері.
